# 1. 项目说明
这一项目是软件开发实训的作业项目，专门针对企业内员工用餐、为企业餐厅开发的订餐外卖系统，员工可通过企业内联网使用该系统，系统用户为企业员工、餐厅员工、餐厅经理、送餐员等。

某企业为了**方便员工用餐**，为餐厅开发了一个订餐外卖系统，企业员工可通过企业内联网使用该系统。系统用户为企业员工、餐厅员工、餐厅经理、送餐员等。
系统的顾客是注册到系统的企业员工，可以查看菜单和今日特价、订餐（如果未登录，需先登录）、支付、预约规律的订餐（每天定时生成订单）、投诉等。
餐厅员工可以进行接受订单、生成付费请求、备餐和请求送餐。
餐厅经理是餐厅管理层，可以管理菜单、查看每日销售情况及简单数据统计、投诉。
送餐员可以选择确认送餐单、确认送餐到达时间、打印送餐小票以及确认已送到。

项目核心流程如下：
1. 企业员工作为普通用户可注册到系统、成为系统的顾客。然后**登录**前台系统，**查看菜单和今日特价菜品**，员工选择菜品进行**订餐**（如果未登录则跳转登录）、**支付**。员工还可**预约订餐**，即每天定时生成订单。如菜品和服务有问题，员工还可进行**投诉**。
2. 餐厅的员工确认订单后，将生成付费请求，开始备餐和请求送餐（生成送餐单）。
3. 餐厅经理是餐厅管理层，可以**管理菜单**、查看**每日销售情况及简单数据统计**、**投诉**。
4. 送餐员可以选择确认送餐单、确认送餐到达时间、打印送餐小票以及确认已送到。

顾客订餐过程如下：
1. 顾客请求查看菜单，系统显示菜单和今日特价；
3. 顾客选菜；
4. 系统显示订单和价格、送餐时间、地点和支付方式；
5. 顾客指定送餐时间、地点和支付方式，确认订单；
8. 餐厅员工确认接受订单，给顾客的界面可显示订单状态（确认中、备餐中、送餐中及预计到达时间），获知相关订餐信息，开始备餐。
9. 备餐完成后，餐厅员工请求送餐、生成送餐单。
10. 送餐员确认并开始送餐，送餐到达后确认已送到，并记录送餐到达时间。

管理员管理过程如下：
1. 管理员在后台系统增加菜品，包括菜品口味数据等。
2. 管理员用菜品发布每日菜单（一次性提前发布一周菜单），同时可设置菜单中菜品的今日特价。
3. 管理员可管理其他餐厅员工和系统用户的信息。

本项目包括前台和后台，前端界面采用若依框架，后端采用SpringBoot开发，数据库使用了MySQL，还使用了RabbitMQ消息队列。

UI界面：
- （餐厅经理）管理员后台界面：员工管理、菜品管理、菜单管理、订单管理、统计信息；
- 餐厅员工（厨师）界面：订单列表界面
- 员工前台界面：菜品界面、订单界面（确认中、备餐中、送餐中……）
- 送餐员界面：送餐列表界面
- 注册登录界面：根据账号不同跳转到管理员后台、餐厅员工、普通员工、送餐员界面。

==划分的模块包括：员工管理、菜品管理、菜单管理、订单管理、信息统计、**送餐管理**、登录管理、**支付系统**等==。

我在这个项目中负责了内容管理、媒资管理、订单支付模块的设计与开发。
	- 内容管理模块，是对平台上的课程进行管理，课程的相关信息比较多。这里在数据库设计了课程基本信息表、课程营销表、课程计划、课程师资表进行存储。
	- 培训机构要发布一门课程，需要填写课程基本信息、课程营销信息、课程计划信息、课程师资信息，填写完毕后需要提交审核，由运营人员进行课程信息的审核，整个审核过程是程序自动审核加人工确认的方式，通常24小时审核完成。
	- 课程审核通过即可发布课程，课程的相关信息会聚合到课程发布表中，这里不仅要将课程信息写到课程发布表，还要将课程信息写到索引库、分布式文件系统中，所以这里存在分布式事务的问题，项目使用本地消息表加任务调度的方式去解决这里的分布式事务，保存数据的最终一致性。

# 6. Maven单体项目启动
创建一个常规的Maven环境，然后导入RuoYi前后端不分离版依赖（需要自己打包），使用ruoyi-admin下面的两个配置文件 `application.yml, application-druid.yml` 稍作修改。因为是SpringBoot项目、且需要若依框架作为脚手架，这里导入 `RuoYiApplication` ：
```java
package com.memcpy0;  
  
import com.ruoyi.RuoYiApplication;  
import lombok.extern.slf4j.Slf4j;  
import org.springframework.boot.SpringApplication;  
import org.springframework.boot.autoconfigure.SpringBootApplication;  
import org.springframework.boot.web.servlet.ServletComponentScan;  
import org.springframework.transaction.annotation.EnableTransactionManagement;  
  
@Slf4j // lombok提供的注解,可以直接使用log变量打印日志  
@SpringBootApplication  
@ServletComponentScan  
@EnableTransactionManagement  
public class RuoyiTakeoutAppApplication {  
    public static void main(String[] args) {  
        SpringApplication.run(  
                new Class[]{RuoYiApplication.class, RuoyiTakeoutAppApplication.class},  
                args  
        );  
        log.info("项目启动成功");  
    }  
}
```
